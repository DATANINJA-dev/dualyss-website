## Description
Provides authoritative CLI design patterns and usability principles based on industry standards. Covers command naming conventions, option standards, UX design patterns, and evaluation frameworks for creating intuitive, consistent command-line interfaces.

Use when:
- Evaluating command naming choices
- Designing new CLI commands
- Auditing CLI usability
- Resolving command structure debates

## When to Use This Skill
- **Command naming decisions**: Should it be `setup`, `init`, `configure`, or `prepare`?
- **Workflow design**: Verb-noun vs noun-verb patterns
- **Option standards**: Which flags should be short vs long?
- **Usability evaluation**: Is the CLI intuitive for first-time users?
- **Framework architecture**: Does the command structure follow industry conventions?
